## Chapter 1: Core Concepts

Imagine your data as a network, not a spreadsheet.

- **Nodes**: Points in the network. Can be People, Companies, Scientific Papers.
  - Example: `(p:Person {name: "Ana"})`
- **Relationships**: Lines connecting the points. They have direction.
  - Example: `Ana -> [:KNOWS] -> Bob`

### Quick Setup (Python)

If you're using Python, loading data looks like this:

```python
import nopaldb

# Start database
graph = nopaldb.Graph.in_memory()
tx = graph.begin_transaction()

# Create nodes
ana = tx.add_node("Person", {"name": "Ana", "age": 28, "city": "NYC"})
bob = tx.add_node("Person", {"name": "Bob", "age": 35, "city": "SF"})
carla = tx.add_node("Person", {"name": "Carla", "age": 22, "city": "LA"})

# Connect (Ana knows Bob, Bob knows Carla)
tx.add_edge(ana, bob, "KNOWS")
tx.add_edge(bob, carla, "KNOWS")

tx.commit()
```

## Chapter 2: Basic Queries

How do we retrieve this info? The structure is simple: `FIND` (what you want) `FROM` (where you look).

### Your First Query
```nql
find p.name, p.age
from (p:Person)
```

**Result:**
```json
{"p.name": "Ana", "p.age": 28}
{"p.name": "Bob", "p.age": 35}
...
```

### Filtering Data (Sociology/Marketing)
Say you want a specific segment: young people in NYC.

```nql
find p.name
from (p:Person)
where p.age < 30 and p.city = "NYC"
```

**Explanation:**
- `(p:Person)`: Search in all nodes labeled Person.
- `where`: Apply logical filter.

## Chapter 3: Discovering Connections

This is where NQL shines. Instead of complicated SQL JOINs, you simply draw the arrow.

### Who knows whom?
```nql
find a.name, b.name
from (a:Person) -> [:KNOWS] -> (b:Person)
where a.name = "Ana"
```
Reads as: "Find people `b` where `a` (Ana) has a `KNOWS` relationship to `b`".

### Influence Analysis (2-step chain)
Imagine analyzing idea propagation. If Ana influences Bob, and Bob influences Carla, whom does Ana influence indirectly?

```nql
find indirect.name
from (ana:Person {name: "Ana"}) -> [:KNOWS] -> (middle) -> [:KNOWS] -> (indirect)
```

**Mental Visualization:**
`Ana -> Middle -> Indirect`

## Chapter 4: Quantitative Analysis

For analysts needing aggregated metrics.

### Counting Results
How many people live in NYC?
```nql
find count(*)
from (p:Person {city: "NYC"})
```

### Basic Statistics
Calculate average age of active users.
```nql
find avg(u.age), min(u.age), max(u.age)
from (u:User)
where u.active = true
```

## Chapter 5: Data Science & Integration

NopalDB is designed to fit into your Machine Learning or Data Analysis pipeline.

### Direct Export (DataFrames)
The most efficient way to move data for ML is not iterating row-by-row, but exporting in bulk.

**NQL Query:**
```nql
find u.age, u.interests
from (u:User)
export csv with path="social_graph.csv", header=true
```

**Python (Pandas):**
```python
import pandas as pd

# Read the file generated by NopalDB
df = pd.read_csv("social_graph.csv")

# Quick analysis
print(df.describe())

# Train model
from sklearn.cluster import KMeans
kmeans = KMeans(n_clusters=3).fit(df[['u.age']])
```

### Advanced Example: Fraud Detection
Identify accounts sharing the same device (IP) across multiple transactions.

```nql
find acct1.id, acct2.id
from (acct1:Account) -> [:USED_IP] -> (ip:IPAddress) <- [:USED_IP] <- (acct2:Account)
where acct1.id != acct2.id
```
*Pattern: Two different accounts pointing to the same IP.*

## Command Summary

| Command | Usage |
|---------|-------|
| `FIND` | Selects properties (`p.name`) or aggregations (`count(*)`). |
| `FROM` | Defines pattern (`(a)->(b)`). |
| `WHERE` | Filters (`age > 18`). |
| `LIMIT` | Restricts results (good for exploration). |
| `EXPORT` | Exports results to CSV/JSON (optional path). |

---

## EXPORT Examples

```nql
find p.name, p.age
from (p:Person)
order by p.age
export csv with path="users.csv", header=true
```

```nql
find p.name, p.age
from (p:Person)
limit 100
export json with jsonl=true
```

Note:
- Recommended syntax places `export` at the end.
- Prefix form (`export ...` before `find`) is not supported.
